## Sobre o Gerenciador de Tarefas

O Gerenciador de Tarefas é um sistema Laravel para gerenciar tarefas no seu servidor. Ele funciona como uma interface para `cronjobs` e executa tarefas em segundo plano, seja um script Linux, um arquivo PHP ou um comando específico no seu servidor de produção. Pode ser utilizado tanto em servidores dedicados quanto compartilhados.

## Instalação

Para instalar, são necessários os seguintes requisitos:

- PHP versão 8.2 ou superior
- Composer versão 2.7 ou superior
- cURL instalado
- Banco de dados: sqlite3, MySQL, PostgreSQL ou MongoDB

### Instalação Passo a Passo

1. Clone este repositório ou faça o download para o seu servidor.
2. Instale as dependências com o Composer:
    ```bash
    composer install --no-interaction --prefer-dist --optimize-autoloader
    ```
3. Crie um arquivo `.env` a partir do arquivo `.env.example`:
    ```bash
    cat .env.example >> .env
    ```
4. Gere uma chave para a aplicação:
    ```bash
    php artisan key:generate
    ```
5. Execute a migração do sistema:
    ```bash
    php artisan migrate
    ```

## Configurando Serviços

Após a instalação, o sistema estará acessível via URL de instalação. Para acessar o painel, use a rota `/manager`.

No entanto, o sistema ainda precisa ser configurado. Para que o sistema funcione, é necessário apontar apenas 1 cronjob para o endpoint `/supervisor` com `cURL` ou `wget`. Dessa forma, o sistema executa internamente o comando `app:supervisor`, que mantém os processos e serviços registrados em execução contínua de acordo com a configuração e o intervalo definido.

> **Nota:** Para garantir que os serviços do Gerenciador permaneçam sempre em execução, recomenda-se usar um intervalo de 1 minuto para o cronjob.
